基于 hexo 的 blog 搭建记录

Posted on 2019-01-15 | In Blog

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

软件安装

安装 Git bash

安装 Node js

安装 hexo

本地部署

创建blog文件夹作为整个项目的根目录，进入blog文件夹，初始化hexo。然后将在blog路径下生成一个hexo项目的结构。

可能需要安装部署工具，否则无法 deploy。

启动本地服务器,然后在 http://localhost:4000 可以预览当前设置下网站的效果。

远程部署

创建 GitHub Page

在GitHub中新建一个repo，名字必须是GitHub的 username.github.io。

连接 GitHub 账户

在 git bash 中生成ssh的key，在windows系统中生成的key默认位于C:\Users\username\.ssh中。将生成的 id_rsa.pub其中的内容复制到GitHub设置中的SSH key的位置。

修改网站的用户名等信息

hexo 连接 GitHub repo，编辑blog目录下的 _config.yml 配置文件，冒号后必须有一个空格。

将网站部署到远程

主题配置

启用next主题

将 Next 主题下载到本地的 theme 文件夹

next 主题的目录中也有一个同名的 _config.yml 配置文件，通常将其称为主题配置文件，而将之前的blog根目录的配置文件称为站点配置文件。

然后将站点配置文件中的theme项修改为next，就将当前的主题设置为next，重新生成，部署之后就可以看到next主题的效果。

next主题本身自带了四种风格，可以在其主题配置文件中搜索 scheme 关键字，然后设置相应的风格。
