C 语法

Posted on 2020-06-25 | In C语言

数据类型

sizeof() 表示不同的数据类型对应的字节数

  • sizeof(char) = 1 byte
  • sizeof(short) = 2 byte
  • sizeof(int) = 1 word
  • sizeof(long) = 1 word
  • sizeof(long long) = 8 byte

输出格式

c语言中无法以二进制的格式输出数字

八进制和十进制只是如何将数字表达为字符串，与计算机内部表示形式无关。

八进制的存在是因为早期的计算机字节是12的倍数。

表示范围

char ：-128 - 127（-2^7 到 -2^7-1）

short ：-32768-32767（-2^15 到 -2^15-1）

int 类型的位宽取决于CPU中寄存器的位宽，因此在不同位宽的计算机上表示的字节数不同。

整形short/int/long/ 类型在计算机内部使用二进制补码存储；

浮点类型double/float 在计算机内部使用编码形式存储，不是自然二进制数，不能直接进行运算。？？？

补码

补码的意义就是补码和原码相加可以得到一个最高位溢出的 “0”。

二进制补码的理解：

补码例子

因为 char 类型只能表示 1 个字节，所以 i 在计算机内部的二进制表示为 1111_1111, 此时最高位表示符号位，因此表示 -1。

而 int 类型可以表示 4 个字节，所以此时 i 在计算机内部的二进制表示为 00000000_00000000_00000000_11111111，最高位为0，表示当前为正数。

显式指定非符号数

  • 可以在数字后面加 u/U 表示 unsigned 或者 l/L 表示long

  • unsinged char i = 255 可以用 i 表示 255 。

  • 使用 unsigned 不是为了扩大一个二进制数能表示的范围，而是为了进行移位运算。

因为 127 = 0111_1111, 127 + 1 = 128 = 1000_0000, 最高位表示符号位。

无符号最高位表示数值

8位无符号所能表示的最大的数为255，加一之后发生溢出，低八位为全 0。

函数

C99 编译器遇到之前未定义，未声明的函数时，会尝试猜测函数所有参数为int类型，返回值为int类型。

当调用函数时，传入参数与函数定义时的参数类型不匹配时，编译器会自动进行类型转换，因此可能会掩盖一些错误。

循环

for和while

  • 任何for循环可以改写成while循环

  • for循环的流程是：

  1. 执行第一个表达式，初始化 i 变量；
  2. 执行第二个表达式，判断是否成立；
    1. 如果成立，跳到下一步；
    2. 如果不成立，跳出循环；
  3. 执行循环体；
  4. 执行第三个表达式；
  5. 跳到第二步，执行第二个表达式；

break和continue

  • continue 语句是指不再执行本次循环中，continue语句下面的循环体；
  • break 语句是指跳出整个循环，但是只能跳出一层循环；

#define

define 进行宏定义

  1. define 后没有分号

  2. define 的内容可以多行，直到遇到注释符号或者一行结束

  3. define 可以定义空的宏，即只有宏，没有值

  4. define 定义带参数的函数宏

    注意事项

    1. 整个宏要加括号，且每个参数都要加括号，否则可能会出错.
      如果 #define func(x) (x) * 2 + 1 , 则 10*func(1)+1 预处理之后变为 10* 1*2+1 +1=22 ,
      而预期要得到的是 10*(1*2+1)+1=31
    2. 参数宏后面最好不加分号

  5. 预定义的宏

#include

插入文件中内容，进行纯文本的替换

误区

  • stdio.h 文件中包含 printf 函数的原型，因此 #include <stdio.h> 只是将 printf函数的原型复制到 #include <stdio.h> 所在的行；

  • 至于 printf 函数的定义则在另外的位置。

  • include头文件的作用只是为了让编译器知道 printf 函数的原型，保证调用时参数类型的正确；

  • 如果没有 #include stdio.h 则编译器默认函数参数类型及返回值类型都是 int 类型；

  • 具体的 printf 函数的实现在另外的地方，在比如某个.lib(windows) 或者 .a(Linux) 库中，现在的c编译器默认都会引入这些标准库的函数实现；

头文件的使用规范

  • 在使用（main函数所在文件）和定义函数的地方都应该 #include 对应的头文件
  • 一般任何 c 文件都有对应的 h 文件，把所有的对外公开的函数原型和全局变量的声明放进去
    • 全局变量可以在不同的C文件之间共享
    • 全局变量前面需要加 extern 关键字，表明可以被外部函数使用
  • static 用法
    • 在函数前面加上static，则该函数只能在其所在的编译单元中使用；
    • 在全局变量前面加上static，则该全局变量只能在其所在编译单元中使用；

标准头文件结构

  • 可以避免头文件的重复引用问题

变量和声明

声明不产生代码，包括：

  • 函数原型
  • 变量/结构/枚举/声明
  • 宏声明
  • 类型声明（typedef）

定义会产生代码，包括：

  • 函数定义
  • 全局变量

初始化是定义做的事情，声明不会做初始化的工作

  • int i; 是变量定义
  • extern int i; 是变量声明
  • 函数定义是定义，函数原型是声明

一般，推荐在头文件中只放声明。

  • 可以避免在不同的编译单元中存在同名的函数
  • 但是某些编译器允许使用weak修饰符来允许存在同名的函数
