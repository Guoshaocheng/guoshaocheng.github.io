vim 正则表达式

Posted on 2018-05-20 | In Linux

模式设置

  • 元字符是具有特殊意义的字符，可以表达自身字符本身以外的含义。

  • 由于 vim 本质上还是一个编辑器，所以 vim 中的元字符与其他的编程语言中的元字符有一些不同，普通模式下的大部分元字符必须加上反斜杠进行转义才能起到元字符的作用。

  • 在 vim 中，不同模式下的元字符的使用方式是不同的。

magic 模式 (\m)

  • 这是 vim 中的默认模式，也是 vim 推荐的用法。在 \m(magic) 模式下，除了三个元字符之外，其他的元字符都要加上反斜杠进行转义。这三个元字符是：
    • $
      • 用于匹配行末
    • ^
      • 用于匹配行首
    • .*
      • 用于匹配任意字符
      • . 匹配的是任意单个的字符，而 * 匹配的是任意的数量。
      • . 也可以与其他表示数量的元字符使用，同样不用加反斜杠。
      • * 也可以与其他表示字符的元字符使用，也不用加反斜杠

nomagic 模式 (\M)

  • \M 模式与 \m 模式很相近，唯一的不同是对待 .* 的区别, \M 模式下将其视为普通的字符。

very magic 模式 (\v)

  • 在此模式下，一切元字符都 不用 加反斜杠，直接使用即可。
  • 一切 并不准确，只有字符类的元字符才可以不加反斜杠，字母类的元字符还是要加上反斜杠。

very nomagic 模式 (\V)

  • 在此模式下，一切元字符都 必须 加反斜杠。

元字符分类

以下皆以 magic 模式下的写法进行说明

匹配字符

.

  • 匹配任意单个字符

[abc]

  • 匹配中括号中任意一个字符

[^abc]

  • 匹配除方括号中字符外的任意一个字符

\d

  • 匹配任意数字，等同于 [0-9]

\D

  • 匹配任意数字之外的字符，等同于 [^0-9]

\w

  • 匹配任意一个数字或字母，等同于 [0-9a-zA-Z]

\W

  • 匹配任意一个字母或数字之外的字符，等同于 [^0-9a-zA-Z]

\s

  • 匹配任意空白字符

\S

  • 匹配任意非空白字符

\x

  • 匹配任意一个十六进制的字符，等同于 [0-9a-fA-F]

\X

  • 匹配任意非一个十六进制的字符，等同于 [^0-9a-fA-F]

匹配数量

匹配优先，即贪婪匹配，尽可能多的匹配

*

  • 0 到任意多个

\+

  • 1 到任意多个

\?

  • 0 个或 1 个

\{n}

  • n 个

\{n,m}

  • n 个到 m 个之间

\{n,}

  • 最少 n 个，但是会尽可能多的去尝试匹配

\{,m}

  • 最多 m 个
  • 忽略优先，即非贪婪匹配，尽可能少的匹配

\{-n,m}

  • 匹配 n 到 m 个，但是匹配到 n 个之后就不会再去尝试匹配下一个字符

\{-,m}

  • 匹配 0 到 m 个，在非贪婪匹配模式下，这种情况会全部匹配，因为可以匹配 0 个，而匹配到 0 个之后就不会再去尝试匹配下一个字符。

\{-n,}

  • 至少匹配 n 个，这种情况下就只会匹配 n 个。

匹配位置

$

  • 匹配行尾

^

  • 匹配行首

\<, \>

  • 匹配单词的开始和结尾

变量捕获

普通捕获

  • 使用 \( 和 \) 括起的匹配内容可以在后面使用 \1, \2 进行捕获。

函数式捕获

  • 在函数式中使用时，可以使用 submatch(1), submatch(2) 来引用 \1， \2 的内容。
  • 函数式的使用方式为 s/str/\=函数式
    • line(".") 表示当前行的行号
    • line("<") 表示当前选中行行首的行号
